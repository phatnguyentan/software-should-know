# 04.22 Push từ server xuống client một chiều

> **Nhóm:** Giai đoạn 4 — Thiết kế giao tiếp
> **Phân nhóm:** Server-Sent Events (SSE)

---

## 1 vấn đề duy nhất

Hiểu đúng **Push từ server xuống client một chiều** để thiết kế và vận hành hệ thống enterprise chắc hơn ở phần **API Design**.

---

## Giải thích nhanh

**Push từ server xuống client một chiều** là khái niệm nói về mô hình server đẩy sự kiện một chiều xuống client qua SSE.

Trong bối cảnh enterprise system design, nó giúp team đưa ra quyết định đúng hơn về api design thay vì chỉ xử lý vấn đề ở mức code cục bộ.

Nó thường được học cùng các khái niệm khác trong nhóm **Server-Sent Events (SSE)** để thấy rõ trade-off và phạm vi áp dụng.

## 1 ví dụ thực tế

Trong một nền tảng commerce có mobile app, partner và service nội bộ, **Push từ server xuống client một chiều** giúp API rõ contract hơn và giảm rủi ro khi client retry, nâng cấp phiên bản hoặc tích hợp ngoài hệ thống.

---

## 3 lỗi phổ biến

1. Thiết kế contract theo màn hình hiện tại thay vì theo resource/use case.
2. Không chuẩn hóa lỗi và hành vi retry.
3. Để consumer phụ thuộc vào chi tiết implementation bên trong.
