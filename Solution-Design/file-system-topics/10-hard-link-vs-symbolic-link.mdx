# 10. Hard link vs symbolic link

> **Nhóm:** Giai đoạn 1 — Cấu trúc lưu trữ cốt lõi

---

## 1 vấn đề duy nhất

Cần chia sẻ dữ liệu mà không copy file, nhưng vẫn phải kiểm soát hành vi khi xóa/di chuyển file gốc.

---

## 1 sơ đồ

```
Hard link: /a/report -> inode 1024 <- /b/report
Symlink:   /c/report -> "/a/report" (path pointer)
```

---

## 1 ví dụ thực tế

Kho artifact tạo hard link cho file release giữa nhiều thư mục để tiết kiệm dung lượng; symlink chỉ dùng cho shortcut cấu hình.

---

## 3 lỗi phổ biến

1. Dùng symlink cho tài liệu quan trọng rồi bị gãy link khi đổi path deploy.
2. Xóa một hard link và tưởng đã giải phóng dung lượng ngay.
3. Không giới hạn quyền ghi trên file được chia sẻ qua hard link.
